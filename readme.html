---- GALVCV_lite

-- INTRO

GALVCV is a program made to heavily automate the process of converting an UTAU voicebank from CV to VCV format. it runs entirely out of a single .bat file and is able to do all but one step on its own with human input.

CV to VCV conversion was revolutionized by Catlione. while it produces amazing results, her method is very time consuming. GALVCV automates her techniques. this program could not exist without her amazing work.

GALVCV_lite is the first iteration of GALVCV and serves as a proof of concept. it has the same features as the full version of GALVCV is planned to have, but uses shortcuts that may lead to less realistic results.

GALVCV is made by galatea (me!). if you have any inquiries, feel free to join the GALVCV discord!

-- DISCLAIMER

GALVCV, while it produces good results, is an automated process (obviously). your computer cannot listen to an audio file and decide if it sounds good or not. please, do not expect GALVCV to produce perfect results; while it's a serious program, it was made for fun. with that said, let's get started!

GALVCV_lite only produces VCV samples. voicebanks made with it do not have V, VV, or CV samples.

-- TERMS

if you're using a voicebank made with GALVCV in a cover, you don't have to give credit - but if you're posting a sample explicitly to showcase the conversion, you must acknowledge GALVCV.

you are more than welcome to explore GALVCV's script and build on it; all that i ask is that you don't reupload your edited script, and i encourage you to share your work in the discord!

---- DEPENDENCIES

GALVCV requires the following:
  • SoX installed and added to PATH
  • moresampler
  • windows computer in japanese locale

---- VOICEBANK PREP

even though i'm perfect and amazing, GALVCV can't do everything. thankfully, it doesn't ask for much.
in order for your voicebank to function the best it can, here's a checklist.
  1. vocal samples are at least half a second long
  2. no more than a little bit of noise before vocals start in the .wav or the vocals start at the very beginning of the .wav*
  3. .wav filenames are nothing but the sample - "a.wav" or "あ.wav".
*for GALVCV_lite, this only needs to be ensured for vowels & n.

it's not a requirement, but i'd reccomend making a backup of your voicebank. GALVCV doesn't override anything, but it's nice to keep a clean version.

---- USAGE

usage for GALVCV is extremely easy. these instructions are partially included in the program itself, but this will serve as further explanation.

1. run GALVCV_lite

2. input the directory of the voicebank you'd like to convert. this directory must be the directory that contains the wav files; if you have multiple pitches in a voicebank, pick a specific one. do not include any quotes.
example: C:\Program Files (x86)\UTAU\voice\uta

you cannot drag and drop the file onto the .exe in GALVCV_lite.

3. input the encoding of the voicebank. 1 for romaji ("a.wav"), 2 for hiragana ("あ.wav")
if you get an error after hitting enter, SoX is not properly installed & added to the windows PATH.

4. at this point, the program will prompt you back here. keep the window open, but do not press anything.

open the ust included with GALVCV. all you should do with this ust is set the singer the voicebank you referenced in step 1, make sure both tools are set to moresampler. do not touch the flags. once the project is initialized, make sure the preutterance and overlap are reset & reset all envelopes so that there's no crossfading.

once everything is initialized, render the ust wherever you'd like.

5. back in the command prompt, press any key to continue the process. input the location of the wav you just rendered, including the wav itself.
example: C:\Users\GALVCV\Desktop\GALVCV_lite.wav

6. input the name for your project; this will be the name of the folder created in the GALVCV directory. you can always rename it afterwards. this isn't very important, just make sure it's identifiable and that you avoid any crazy characters.

7. input the suffix you'd like to add to the oto.ini. this is only needed if you're converting multiple pitches or expressions. if you're not, just hit enter.

this is the longest part, so just wait a bit & look at the cute cat =^..^=

8. at this point, you have four options on how to prep your voicebank for usage. click an option for more details & instructions.

option 1 incorporates vcv strings into an existing voicebank folder. i reccomend this the most, as GALVCV_lite doesn't generate V, VV or CV samples.

input the existing voicebank directory. this should be the root directory of the voicebank, where you can find things like the readme.txt.

you'll be asked if you want to delete your project folder - remember, this is the folder in the GALVCV_lite directory. this is up to you, but this process can't be undone.

after completing the process, there will be a new subfolder in your voicebank that contains the vcv samples. by default it's named after the GALVCV project, but you're free to rename it.

option 2 makes the voicebank project folder a standalone voicebank using placeholder assets. requires no extra input, just make sure you replace the icon, readme.txt, and character.txt.


option 3 makes the voicebank project folder a standalone voicebank using an existing voicebank's assets.

input the existing voicebank directory. this should be the root directory of the voicebank, where you can find things like the readme.txt. after completing the process, i would reccomend changing the voicebank name to differentiate it from the original.

option 4 does nothing to prep the voicebank and leaves the project folder as it is.

after completing preparation, you're done! have fun with your converted voicebank!

---- UTAU USAGE TIPS

GALVCV voicebanks are obviously far from perfect, so here's a few tips to make your output smoother.

- edit the oto.ini. GALVCV generates a very bare-bones base oto. it should be fine, but if you're having offset problems, this should fix it.

- set the consonant velocity of vcv samples to a low number like 50. because GALVCV's generated samples are very short, this can help smooth transitions out.

- create vowel blend aliases using the original CV voicebank's vowels. because GALVCV_lite doesn't generate VV, this is the only way to get smooth vowel transitions. you can do this simply by creating a new alias, dragging the starting point in the middle of the sample, and extending the preutterance and overlap.

- if your voicebank doesn't sound good with long notes, try using ameya's doppeltler resampler.
